What To Watch

With more streaming options than ever, picking a movie for movie night can be a daunting task. What To Watch aims to help solved this problem by providing users with a list of movie recommendations and their available streaming services. I built it using React to create the interface, Redux to manage state, and a Rails API for the back end. I also implemented Material UI to aid in the styling of the site.

Rendezvous

Rendezvous is a single-page application dedicated to crowdsourcing event information. Users can find upcoming events recommended by the community and add their own. It's people-driven and works best when members are actively adding and updating events. Similar to What to Watch, I created a Rails API to store data on the backend. The front end is a pure JavaScript SPA, with custom CSS for styling.

Green Light 2.0

Green Light 2.0 reimagines the functionality of its predecessor. Designed for TV and film development teams, it keeps a running list of scripts and their associated formats (ie, half-hour comedy pilots, feature films, etc.) that the team has read. Built with Rails and custom CSS, it utilizes CRUD, nested forms, and nested routes throughout the project so that users are able to add, edit, and delete scripts.

Harry Potter CLI

The Harry Potter CLI is a Command Line Interface application written in Ruby. Users are able to select a character from Harry Potter and the program will return the name of the actor who plays them in the films. This data is loaded in using the rest-client gem to access an external API containing Harry Potter information. In order to make it more readable, I utilized the color_text gem to make different sections of text different colors.
